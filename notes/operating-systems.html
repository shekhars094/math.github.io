Operating Systems

Process Model

A process is a running program instance with its own address space and resources.

States

  • new → ready → running → blocked → terminated

Context Switch

Switching CPU from one process to another; saves/restores registers, PC, stack pointer, etc.

Threads

Lightweight units of execution within a process sharing address space.

Benefits

  • Concurrency
  • Resource sharing
  • Lower context-switch cost vs processes

CPU Scheduling

Common Algorithms

  • FCFS, SJF, SRTF, RR (time quantum), Priority

Memory Management

Paging

Virtual memory divided into pages; physical memory into frames; uses page tables and TLB.

Segmentation

Memory as logical segments (code, data, stack) with base/limit.

File Systems

Inodes & Directories

Metadata (inode) plus directory entries mapping names to inodes.

Synchronization

Primitives

  • Mutex, Semaphore, Monitor, Condition Variable

Deadlocks

Conditions

  • Mutual exclusion, Hold & wait, No preemption, Circular wait
